Hexo

What it is?

A fast, simple & powerful blog framework, powered by Node.js.^1

  • Therefore, you need to install Node.js first in order to install hexo.

You write posts in Markdown (or other markup languages) and Hexo generates static files with a beautiful theme in seconds.^2

  • I use Markdown + VS Code.

Installation

  • You can use hexo -v to check if the installation succeeds.

Setup your blog

  1. Prepare yourself a folder for your project (If something goes wrong, don’t worry, just rm the whole folder).
  2. Use cd to go to the folder.
  3. Use hexo init to setup the blog.
  4. Use hexo server to start the server and you can preview your blog on your local machine.

Change Configuration

  1. My lazy way here is to use vim _config.yml and of course you can use any other editors. The main point is about this file, namely _config.yml. For example, I changed the author and title in it.
  2. Don’t forget to regenerate static files using hexo generate when the changes is about static files such as your blog style. Otherwise your personal changes don’t apply to your blog. When you try to deploy, git will tell you nothing to be committed. You can also use hexo clean before the generation to clean the static files.

Deploy your blog

I use Github Pages and I think it is the easiest one.

  1. Install Git.
  2. Install hexo-deployer-git.
  3. Create a new repository username.github.io. It must be exactly your username!
  4. Setup _config.yml
  5. Deploy using hexo deploy or hexo deploy -g. The later command means to generate before deployment so you can avoid some trouble such as what I mentioned before.
